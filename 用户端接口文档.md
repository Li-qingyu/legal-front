# 法律咨询平台用户端接口文档

## 1. 接口基础信息

- 基础路径：`/user`
- 响应格式：
  ```json
  {
    "code": 1, // 1表示成功，0表示失败
    "msg": "操作成功", // 错误信息
    "data": {} // 返回数据
  }
  ```

## 2. 接口列表

| 接口名称 | 请求方法 | 请求URL | 功能描述 |
| --- | --- | --- | ---
| 轮播图接口 | GET | /user/home/carousel | 获取首页轮播图数据 |
| 热门法律案例接口 | GET | /user/home/cases | 获取热门法律案例列表 |
| 法律条文接口 | GET | /user/home/laws | 获取法律条文列表 |
| 搜索接口 | GET | /user/home/search | 搜索法律相关内容 |
| AI法律咨询接口 | POST | /user/home/ai-consult | 提供AI法律问题咨询服务 |

- **请求方法**: GET
- **请求URL**: `/user/home/carousel`
- **请求参数**: 无

```json

  "code": 1,
  "msg": "查询成功",
  "data": [
    {
      "id": 1,
      "title": "专业法律咨询服务",
```

## 4. 通用响应格式

所有接口的响应格式统一为：

```json
{
  "code": 1, // 1表示成功，0表示失败
  "msg": "操作成功", // 错误信息
  "data": {} // 返回数据
}
```

### 状态码说明
- `1`: 请求成功
- `0`: 请求失败
- `400`: 请求参数错误
- `401`: 未授权
- `403`: 禁止访问
- `404`: 资源不存在
- `500`: 服务器内部错误

## 5. 接口规范

1. **请求方法**：
   - GET：用于获取数据
   - POST：用于创建数据
   - PUT：用于更新数据
   - DELETE：用于删除数据
2. **参数传递**：
   - 查询参数：用于GET请求的筛选和分页
   - 请求体：用于POST和PUT请求的数据传递
   - 路径参数：用于标识资源ID
3. **响应格式**：
   - 统一使用JSON格式
   - 包含code、msg、data三个字段
   - code为1表示成功，0表示失败
   - msg为操作结果的描述
   - data为返回的数据
4. **错误处理**：
   - 400：请求参数错误
   - 401：未授权
   - 403：禁止访问
   - 404：资源不存在
   - 500：服务器内部错误
5. **认证机制**：
   - 使用JWT进行身份认证
   - 登录成功后返回token
   - 后续请求需要在请求头中携带token：`Authorization: Bearer <token>`

## 6. 前端调用示例

### 6.1 使用axios调用接口

```javascript
import axios from 'axios';

// 获取轮播图数据
async function getCarousel() {
  try {
    const response = await axios.get('/user/home/carousel');
    return response.data.data;
  } catch (error) {
    console.error('获取轮播图失败:', error);
    return [];
  }
}

// 获取热门法律案例
async function getHotCases() {
  try {
    const response = await axios.get('/user/home/cases', {
      params: {
        page: 1,
        pageSize: 3
      }
    });
    return response.data.data.records;
  } catch (error) {
    console.error('获取热门案例失败:', error);
    return [];
  }
}

// 获取法律条文
async function getLaws() {
  try {
    const response = await axios.get('/user/home/laws', {
      params: {
        page: 1,
        pageSize: 4
      }
    });
    return response.data.data.records;
  } catch (error) {
    console.error('获取法律条文失败:', error);
    return [];
  }
}

// 搜索法律相关内容
async function searchLawContent(keyword) {
  try {
    const response = await axios.get('/user/home/search', {
      params: {
        keyword,
        type: 'all'
      }
    });
    return response.data.data;
  } catch (error) {
    console.error('搜索失败:', error);
    return { cases: [], laws: [], total: 0 };
  }
}

// AI法律咨询
async function aiConsult(question) {
  try {
    const response = await axios.post('/user/home/ai-consult', {
      question
    });
    return response.data.data;
  } catch (error) {
    console.error('AI咨询失败:', error);
    return null;
  }
}
```

### 6.2 在Vue组件中使用

```vue
<script setup>
import { ref, onMounted } from 'vue';
import axios from 'axios';

const carouselData = ref([]);
const cases = ref([]);
const laws = ref([]);
const searchQuery = ref('');

// 初始化数据
onMounted(async () => {
  await loadHomeData();
});

// 加载首页数据
async function loadHomeData() {
  // 获取轮播图
  const carouselResponse = await axios.get('/user/home/carousel');
  carouselData.value = carouselResponse.data.data;
  
  // 获取热门案例
  const casesResponse = await axios.get('/user/home/cases', {
    params: { page: 1, pageSize: 3 }
  });
  cases.value = casesResponse.data.data.records;
  
  // 获取法律条文
  const lawsResponse = await axios.get('/user/home/laws', {
    params: { page: 1, pageSize: 4 }
  });
  laws.value = lawsResponse.data.data.records;
}

// 搜索
async function handleSearch() {
  if (!searchQuery.value) return;
  
  const response = await axios.get('/user/home/search', {
    params: { keyword: searchQuery.value }
  });
  
  // 处理搜索结果
  console.log('搜索结果:', response.data.data);
}

// AI咨询
async function handleAiConsult(question) {
  const response = await axios.post('/user/home/ai-consult', {
    question
  });
  
  // 处理AI回答
  console.log('AI回答:', response.data.data);
}
</script>
```

## 7. 接口安全

- 所有接口应使用HTTPS协议
- 敏感操作应进行身份验证
- 接口请求应进行参数校验
- 防止SQL注入等安全问题

## 8. 接口性能

- 对于高频请求的接口，建议实现缓存机制
- 分页查询应合理设置默认页大小
- 对于AI咨询等耗时操作，可考虑使用异步处理

## 9. 版本控制

- 接口版本建议通过URL路径进行管理，如 `/user/v1/home/carousel`
- 当接口发生破坏性变更时，应升级版本号

## 10. 错误处理

- 服务端应返回明确的错误信息
- 前端应根据错误码进行相应的错误处理和用户提示

## 11. 总结

本接口文档设计了法律咨询平台用户端所需的全部API接口，包括轮播图、热门法律案例、法律条文、搜索和AI法律咨询功能。这些接口将支持前端页面的正常运行，为用户提供良好的使用体验。

接口设计遵循了RESTful风格，响应格式统一，参数说明清晰，便于前端开发人员理解和使用。同时，文档提供了详细的前端调用示例，帮助开发人员快速集成这些接口。
