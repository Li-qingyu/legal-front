# 用户端个人中心接口文档

## 接口概述

用户端个人中心页面主要涉及以下两个接口：
1. 查询用户信息接口
2. 更新用户信息接口

## 1. 查询用户信息接口

### 接口描述
根据用户ID查询用户详细信息

### 请求信息
- **请求方式**: `GET`
- **请求路径**: `/user/info/{id}`
- **请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Integer | 是 | 用户ID |

### 请求示例
```http
GET /api/user/info/1
```

### 响应数据
```json
{
  "code": 1,
  "msg": "查询成功",
  "data": {
    "id": 1,
    "username": "张三",
    "email": "zhangsan@example.com",
    "phone": "13800138000",
    "status": 1,
    "createTime": "2026-01-01 10:00:00",
    "updateTime": "2026-01-01 10:00:00"
  }
}
```

### 响应参数说明
| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码，1表示成功，0表示失败 |
| msg | String | 返回消息 |
| data | Object | 用户信息对象 |
| data.id | Integer | 用户ID |
| data.username | String | 用户名 |
| data.email | String | 邮箱 |
| data.phone | String | 手机号 |
| data.status | Integer | 用户状态，1表示正常，0表示禁用 |
| data.createTime | String | 创建时间 |
| data.updateTime | String | 更新时间 |

## 2. 更新用户信息接口

### 接口描述
更新用户基本信息或修改密码

### 请求信息
- **请求方式**: `POST`
- **请求路径**: `/user/update`
- **请求头**: `Content-Type: application/json`
- **请求参数**:

#### 2.1 更新基本信息
```json
{
  "id": 1,
  "username": "张三",
  "email": "zhangsan@example.com",
  "phone": "13800138000"
}
```

#### 2.2 修改密码
```json
{
  "id": 1,
  "oldPassword": "old123456",
  "newPassword": "new123456"
}
```

### 请求参数说明
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Integer | 是 | 用户ID |
| username | String | 否 | 用户名（更新基本信息时必填） |
| email | String | 否 | 邮箱（更新基本信息时必填） |
| phone | String | 否 | 手机号（更新基本信息时必填） |
| oldPassword | String | 否 | 旧密码（修改密码时必填） |
| newPassword | String | 否 | 新密码（修改密码时必填） |

### 响应数据
```json
{
  "code": 1,
  "msg": "更新成功",
  "data": null
}
```

### 响应参数说明
| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码，1表示成功，0表示失败 |
| msg | String | 返回消息 |
| data | Object | 返回数据，通常为null |

## 前端调用示例

### 1. 查询用户信息
```javascript
import { queryInfoApi } from '@/api/user';

const loadUserInfo = async () => {
  try {
    const loginUser = JSON.parse(localStorage.getItem('loginUser'));
    if (loginUser && loginUser.id) {
      const result = await queryInfoApi(loginUser.id);
      if (result.code) {
        userInfo.value = result.data;
      } else {
        ElMessage.error(result.msg || '获取用户信息失败');
      }
    }
  } catch (error) {
    ElMessage.error('获取用户信息失败');
    console.error('获取用户信息失败:', error);
  }
};
```

### 2. 更新用户信息
```javascript
import { updateApi } from '@/api/user';

const handleEdit = async () => {
  try {
    const result = await updateApi({
      id: userInfo.value.id,
      username: editForm.value.username,
      email: editForm.value.email,
      phone: editForm.value.phone
    });
    if (result.code) {
      ElMessage.success('修改成功');
      await loadUserInfo();
    } else {
      ElMessage.error(result.msg || '修改失败');
    }
  } catch (error) {
    ElMessage.error('修改失败');
    console.error('修改失败:', error);
  }
};
```

### 3. 修改密码
```javascript
import { updateApi } from '@/api/user';

const handleChangePassword = async () => {
  try {
    const result = await updateApi({
      id: userInfo.value.id,
      oldPassword: passwordForm.value.oldPassword,
      newPassword: passwordForm.value.newPassword
    });
    if (result.code) {
      ElMessage.success('密码修改成功');
      passwordDialogVisible.value = false;
    } else {
      ElMessage.error(result.msg || '修改失败');
    }
  } catch (error) {
    ElMessage.error('修改失败');
    console.error('修改失败:', error);
  }
};
```

## 表单验证规则

### 基本信息验证
```javascript
const editRules = {
  username: [
    { required: true, message: '用户名为必填项', trigger: 'blur' },
    { min: 2, max: 20, message: '用户名长度为2-20个字', trigger: 'blur' }
  ],
  email: [
    { required: true, message: '邮箱为必填项', trigger: 'blur' },
    { type: 'email', message: '请输入合法的邮箱地址', trigger: 'blur' }
  ],
  phone: [
    { required: true, message: '手机号为必填项', trigger: 'blur' },
    { pattern: /^1[3-9]\d{9}$/g, message: '请输入合法的手机号', trigger: 'blur' }
  ]
};
```

### 密码验证
```javascript
const passwordRules = {
  oldPassword: [
    { required: true, message: '请输入旧密码', trigger: 'blur' }
  ],
  newPassword: [
    { required: true, message: '请输入新密码', trigger: 'blur' },
    { min: 8, message: '密码长度至少8位', trigger: 'blur' },
    { pattern: /^(?=.*[A-Za-z])(?=.*\d)[A-Za-z\d]{8,}$/, message: '密码必须包含字母和数字', trigger: 'blur' }
  ],
  confirmPassword: [
    { required: true, message: '请确认新密码', trigger: 'blur' },
    {
      validator: (rule, value, callback) => {
        if (value !== passwordForm.value.newPassword) {
          callback(new Error('两次输入的密码不一致'));
        } else {
          callback();
        }
      },
      trigger: 'blur'
    }
  ]
};
```

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 0 | 操作失败 |
| 1 | 操作成功 |
| 401 | 未登录或登录已过期 |
| 403 | 权限不足 |
| 500 | 服务器内部错误 |

## 注意事项

1. 所有接口都需要在请求头中携带token（如果已登录）
2. 修改密码时需要验证旧密码的正确性
3. 用户名、邮箱、手机号不能为空
4. 密码必须包含字母和数字，长度至少8位
5. 两次输入的新密码必须一致
6. 用户状态为0（禁用）时无法修改信息

## 页面路由

- **路由路径**: `/user/profile`
- **路由名称**: `userProfile`
- **访问权限**: 需要登录，普通用户(role=0)和管理员(role=1)都可访问

## 样式说明

个人中心页面采用与用户端首页一致的样式风格：
- 主色调：蓝色渐变 (#1e3c72 到 #2a5298)
- 辅助色：金色 (#d4af37)
- 背景色：浅蓝渐变 (#f5f7fa 到 #c3cfe2)
- 卡片样式：白色背景，圆角12px，阴影效果
- 响应式设计：支持移动端和桌面端
