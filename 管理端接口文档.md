# 管理端接口文档

## 1. 接口基础信息

- 基础路径：`/admin`
- 响应格式：
  ```json
  {
    "code": 1, // 1表示成功，0表示失败
    "msg": "操作成功", // 错误信息
    "data": {} // 返回数据
  }
  ```

## 7. 接口规范

1. **请求方法**：
   - GET：用于获取数据
   - POST：用于创建数据
   - PUT：用于更新数据
   - DELETE：用于删除数据

2. **参数传递**：
   - 查询参数：用于GET请求的筛选和分页
   - 请求体：用于POST和PUT请求的数据传递
   - 路径参数：用于标识资源ID

3. **响应格式**：
   - 统一使用JSON格式
   - 包含code、msg、data三个字段
   - code为1表示成功，0表示失败
   - msg为操作结果的描述
   - data为返回的数据

4. **错误处理**：
   - 400：请求参数错误
   - 401：未授权
   - 403：禁止访问
   - 404：资源不存在
   - 500：服务器内部错误

5. **认证机制**：
   - 使用JWT进行身份认证
   - 登录成功后返回token
   - 后续请求需要在请求头中携带token：`Authorization: Bearer <token>`

```javascript
// 导入axios实例
import request from '@/utils/request';

// 登录
const login = async (data) => {
  return request.post('/login', data);

// 分页查询用户

// 更新用户

};

// 删除用户

};
```

### 8.2 后端实现示例

```java
// 用户控制器示例
@RestController
@RequestMapping("/admin/users")
public class UserController {
    
    @Autowired
    private UserService userService;
    
    @GetMapping
    public Result queryPage(@RequestParam String name, @RequestParam int page, @RequestParam int pageSize) {
        Page<User> userPage = userService.queryPage(name, page, pageSize);
        return Result.success(userPage);
    }
    
    @GetMapping("/{id}")
    public Result queryInfo(@PathVariable Long id) {
        User user = userService.queryInfo(id);
        return Result.success(user);
    }
    
    @PostMapping
    public Result add(@RequestBody User user) {
        userService.add(user);
        return Result.success();
    }
    
    @PutMapping("/{id}")
    public Result update(@PathVariable Long id, @RequestBody User user) {
        userService.update(id, user);
        return Result.success();
    }
    
    @DeleteMapping("/{id}")
    public Result delete(@PathVariable String id) {
```
